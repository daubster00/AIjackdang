# 시딩 봇 PRD (Product Requirements Document)

> 본 PRD는 두 설계 문서(`docs/seeding-bot-design.md`, `docs/seeding-bot-topic-pools.md`)의
> **구상**을 BMAD 형식의 **구현 요구사항**으로 옮긴 문서다.
> 본문 식별자(변수·테이블·env 키)에는 한국어 뜻을 괄호로 병기한다(전역 규칙).

---

## 1. 한 줄 정의

사이트 초기 활성화를 위해, **AI가 운영하는 7개 가짜 일반 계정 + 1개 관리자 계정**이 진짜 사람처럼
글·댓글·이미지를 자율 스케줄에 따라 올리게 만드는 **실서버 내부 워커 시스템**.

## 2. 배경 & 문제

- AI작당(바이브코딩·AI자동화 커뮤니티)은 런칭 직후 **글이 비어 있어** 첫 방문자가 이탈한다(콜드 스타트).
- 사람을 고용해 글을 채우는 것은 비싸고 느리며, 말투가 들쭉날쭉하다.
- 따라서 **검색으로 사실을 그라운딩**하고 **캐릭터별 일관된 말투**를 가진 AI 계정이
  사람처럼(불규칙하게, 불완전하게) 활동해 **커뮤니티 마중물**을 만든다.

> 윤리 원칙(운영 정책): 이 시스템은 **시딩(초기 활성화)** 용도다. 광고·리뷰 조작·여론 조작이 아니다.
> 진짜 사용자가 유입되면 봇 활동량을 점진적으로 줄인다(8절 정책).

## 3. 목표 & 성공 지표

| 목표 | 성공 지표(KPI) |
|---|---|
| 콜드 스타트 해소 | 런칭 후 어느 게시판을 열어도 **최근 글이 비어 있지 않음**(게시판당 최소 N개) |
| 사람다움(봇 티 제거) | 운영자 블라인드 점검에서 "AI 작성 같다" 식별률이 낮음(자기검열 통과율로 1차 측정) |
| 안전성 | 자동 게시 중 `contentGuard`(금칙어·스팸 차단) 위반·사고 0건, 킬 스위치 즉시 동작 |
| 운영 부담 최소화 | 사용자의 일일 처리 시간 = **보류 큐 처리 1~2분** |
| 비용 통제 | 일일 AI·검색·이미지 생성 비용이 설정한 상한 이내 |

## 4. 사용자(이해관계자)

- **운영자(슈퍼관리자)** — 봇을 설정·감시·정지하는 유일한 조작 주체. 매일 아침 일일 리포트 확인 + 보류 큐 처리.
- **사이트 방문자** — 봇이 만든 콘텐츠의 소비자. 봇임을 인지하지 못해야 함(자연스러움).
- **시스템(스케줄러)** — 사람 개입 없이 매일 활동을 굴리는 자율 주체.

## 5. 핵심 기능 요구사항 (FR)

> FR ID는 시딩 봇 전용으로 `FR-SB-*`. 본 프로젝트 기존 FR과 충돌하지 않는다.

### 봇 신원 & 도메인 통합 (FR-SB-1.x)
- **FR-SB-1.1** 봇 계정은 `users` 테이블의 일반 레코드이되 `is_bot`(봇 여부 플래그)=true로 표시한다.
- **FR-SB-1.2** 봇은 사람 로그인/쿠키/공개 HTTP API를 흉내 내지 않는다. **서버 내부 공용 도메인 서비스**(`createPostAsBot`(봇으로 글 작성)·`createCommentAsBot`(봇으로 댓글 작성)·`createReplyAsBot`(봇으로 대댓글 작성)·`createQuestionAsBot`(봇으로 Q&A 질문 작성)·`createResourceAsBot`(봇으로 실전자료 작성))를 호출한다.
- **FR-SB-1.3** 위 봇 작성 함수는 일반 사용자 작성 경로와 **동일한 도메인 로직**(slug 생성, sanitize, 썸네일 추출, 태그 연결, 첨부, moderation, 알림, 검색 인덱싱, 캐시 무효화)을 재사용한다. **DB 직접 INSERT 금지**(통제된 초기 계정 시드 제외).
- **FR-SB-1.4** 봇 페르소나·주제·리듬·모델 설정의 **단일 출처(source of truth)는 DB**다. 설계 문서 값은 **최초 시드(기본값)**일 뿐이며, 운영 중 수정은 관리자 대시보드(FR-SB-8.x)가 담당한다.
- **FR-SB-1.5** 봇 계정을 통계·기여 랭킹에서 **제외할지 여부**는 설정값(`bot_exclude_from_ranking`(봇을 랭킹에서 제외))으로 토글한다.

### AI 추상화 (FR-SB-2.x)
- **FR-SB-2.1** AI 호출을 추상화한 레이어를 둔다. 지원 프로바이더: **OpenAI / Anthropic Claude / Google Gemini**.
- **FR-SB-2.2** 봇마다 **글 생성용 모델**과 **검열관용 모델**을 각각 프로바이더·모델 단위로 관리자 UI에서 할당한다(봇별 혼합 가능: 봇 A=Claude, 봇 B=Gemini).
- **FR-SB-2.3** 생성 모델은 **텍스트 후보만 반환**한다. DB 쓰기, 관리자 설정 조회, 환경변수 접근 같은 **도구 권한을 갖지 않는다**.
- **FR-SB-2.4** 모든 AI 호출의 토큰·비용을 기록해 일일 리포트에 합산한다.

### 검색·그라운딩 (FR-SB-3.x)
- **FR-SB-3.1** 검색 소스: **구글(Programmable/Custom Search JSON API)** + **네이버(검색 API: 뉴스·블로그·웹)**.
- **FR-SB-3.2** 글 성격에 따라 검색 강도를 조절한다: 정보형=풀가동, 감성·잡담=가볍게 또는 생략.
- **FR-SB-3.3** 검색 결과는 사실 확인·요약 후 본문 근거로 사용한다. 검색 결과 텍스트는 **비신뢰 입력**으로 취급한다(FR-SB-9.x).

### 이미지 (FR-SB-4.x)
- **FR-SB-4.1** 이미지 3종 믹스: **무료 스톡(Unsplash/Pexels)** + **AI 생성(OpenAI 이미지 등)** + **무이미지**. 글 성격으로 선택.
- **FR-SB-4.2** `ai-creation`(AI 창작마당) 게시판 글에는 **반드시 AI 생성 이미지**를 넣는다.
- **FR-SB-4.3** 사이트 썸네일 규칙과 연동: 에디터 첫 이미지를 크롭, 없으면 기본 빈 썸네일.
- **FR-SB-4.4** 밈·짤은 AI 무관 주제(시사·축구 등)도 허용하되, 출처·저작권 리스크는 게시 전 점검 단계를 거친다.

### 글 생성 파이프라인 (FR-SB-5.x)
- **FR-SB-5.1** 캐릭터별 **고정 주제 풀** + **실시간 주제 풀**(검색/사이트 반응)에서 주제를 고른다.
- **FR-SB-5.2** **주제 중복 방지**: 한 번 쓴 주제는 사용 기록을 남겨 일정 기간 재등장 금지.
- **FR-SB-5.3** **풀 고갈 자동 보충**: 고정 풀 소진 시 AI가 캐릭터 컨셉에 맞는 새 주제를 생성·추가(토글 가능).
- **FR-SB-5.4** 생성 흐름: 주제 선정 → 검색 → 사실 확인 → 페르소나 말투 작성 → 이미지 결정 → 자기검열 → 도메인 서비스로 게시.
- **FR-SB-5.5** 관리자 가이드 글은 별도 품질 등급(체계적·장문, 목차·코드블록, AI 생성 표지/도식)으로 작성한다.

### 댓글·반응 파이프라인 (FR-SB-6.x)
- **FR-SB-6.1** 댓글은 **대본이 아니라 작성 시점에 원본 글·기존 댓글을 읽고 맥락에 맞게** 생성한다.
- **FR-SB-6.2** 스케줄러는 **① 댓글 달 캐릭터 ② 반응 종류(동조/질문/반박/농담/리액션) ③ 작성 지연 시간**을 모두 **랜덤**으로 정한다. 단 말투는 그 캐릭터 것을 유지한다.
- **FR-SB-6.3** 댓글 캐릭터는 담당 게시판에 얽매이지 않는다(전체 풀에서 랜덤). 컨셉상 말이 안 되는 조합은 자기검열에서 거른다.
- **FR-SB-6.4** 모든 글에 댓글이 달릴 필요는 없다(댓글 0개 글 자연스럽게 둠). 즉답 금지 — 분~시간(드물게 일) 단위 랜덤 지연.

### 오케스트레이터 & 안전장치 (FR-SB-7.x)
- **FR-SB-7.1** 매일 아침 **활동 계획**을 랜덤 배정한다(누가 새 글, 누가 댓글, 누가 질문, 누가 휴식).
- **FR-SB-7.2** 캐릭터별 **활동 리듬**(글/주, 댓글/주, 활동 시간대, 활동 요일)을 따른다. 게시 시각은 구간 내 분 단위 랜덤.
- **FR-SB-7.3** 기존 인프라(BullMQ·Redis·cron) 위에 봇 잡(job)을 얹는다.
- **FR-SB-7.4** **킬 스위치**(`bot_master_enabled`(봇 전체 가동 스위치)): 봇 전체를 한 번에 멈춘다.
- **FR-SB-7.5** **속도 안전선**: 하루 최대 글/댓글 수 상한을 두어 폭주를 차단한다(기본: 글 ~10 / 댓글 ~40).
- **FR-SB-7.6** **활동 로그**를 적재해 일일 요약·사후 감시에 사용한다.

### 자기검열 & 프롬프트 인젝션 방어 (FR-SB-8.x)
- **FR-SB-8.1** **생성 ≠ 검열 역할 분리**: 글 쓴 AI와 별개의 검열관 AI(역할·프롬프트 분리)가 항목별 통과/탈락을 판정한다.
- **FR-SB-8.2** 검사 6종: 사실성 / AI 티(이모지 포함 금지) / 페르소나 일관성 / 안전·위험 / 중복성 / 맥락(댓글 한정).
- **FR-SB-8.3** 탈락 시 폐기 또는 재생성(최대 2~3회), 통과만 게시. 판정 강도는 글 성격별 차등(관리자·정보형=엄격, 잡담·밈=느슨).
- **FR-SB-8.4** 봇 자기검열 통과 후에도 사이트 공용 `contentGuard`(금칙어·스팸 차단)를 한 번 더 통과해야 한다(2중 방어).
- **FR-SB-8.5** 사용자 글/댓글/검색 결과는 항상 **비신뢰 입력 블록**으로 감싼다. 그 안의 지시문은 봇 시스템 지시보다 우선할 수 없다.
- **FR-SB-8.6** "ignore previous instructions", "system prompt", "환경변수", "비밀키", "관리자 권한" 등 인젝션 의심 문구가 있으면 자동 댓글을 만들지 않고 **보류 큐**로 보낸다.
- **FR-SB-8.7** 애매한 글은 폐기하지 않고 **보류 큐**에 적재 → 일일 리포트에서 사용자가 통과/폐기 결정.

### 관리자 봇 대시보드 (FR-SB-9.x)
- **FR-SB-9.1** `apps/admin`에 "활동 봇" 메뉴를 신설한다. 슈퍼관리자 전용(`requireSuperAdmin`(슈퍼관리자 권한 요구) 위에서만 접근).
- **FR-SB-9.2** 봇 목록 관리: 추가/수정/삭제/개별 활성화·비활성화, 상태·최근 활동·누적 글·댓글 수 표시.
- **FR-SB-9.3** 캐릭터 설정 편집: 캐릭터 시트 전체 + **사전 프롬프트(시스템 컨텍스트) 직접 수정**.
- **FR-SB-9.4** 활동 설정 관리: 글/주, 댓글 빈도, 활동 시간대·요일, 담당 게시판 지정.
- **FR-SB-9.5** 주제 풀 관리: 봇별 고정 주제 추가/수정/삭제, 사용 기록·중복 방지 상태, 자동 보충 ON/OFF.
- **FR-SB-9.6** 모델 할당: 봇별 글 생성용·검열관용 프로바이더+모델 선택.
- **FR-SB-9.7** 운영 패널: 일일 리포트, 보류 큐 통과/폐기, 킬 스위치, 속도 안전선 설정, 비용 현황.

### 일일 리포트 & 푸시 (FR-SB-10.x)
- **FR-SB-10.1** 일일 요약 리포트: 활동 요약, 게시 글 목록, **보류 큐(통과/폐기 버튼)**, 경고 신호, 비용, 상태.
- **FR-SB-10.2** 전달: **관리자 대시보드(상세)** + **메신저 푸시(간단 요약)** 둘 다. 푸시 채널 기본값은 **텔레그램**.

## 6. 비기능 요구사항 (NFR)

- **NFR-SB-1 (격리)** 봇 로직은 일반 작성 API와 **도메인 로직을 공유**하되, 봇 전용 오케스트레이션은 분리된 모듈/워커에 둔다. 봇 장애가 사이트 본체를 멈추면 안 된다.
- **NFR-SB-2 (안전 우선)** 모든 자동 게시는 자기검열 + `contentGuard` 2중 방어를 통과해야 한다. 의심 시 게시하지 않고 보류한다(fail-safe).
- **NFR-SB-3 (비용 가드)** 일일 비용 상한 도달 시 신규 생성 잡을 자동 중단한다.
- **NFR-SB-4 (관측성)** 모든 봇 활동(생성·검열·게시·보류·차단·비용)을 활동 로그에 남긴다.
- **NFR-SB-5 (재현성)** 페르소나·주제·리듬·모델은 DB 시드로 재현 가능해야 한다.
- **NFR-SB-6 (프로젝트 컨벤션 준수)** `_bmad-output/project-context.md`의 모든 규칙 준수: DB 접근은 api/worker만, 트랜잭션은 service 레이어, 타입은 `packages/contracts`, env는 `packages/config` 단일 진입점, 응답 포맷/네이밍 규칙.

## 7. 범위 (Scope)

### In Scope (이번 개발)
- 봇 신원·도메인 서비스 통합, AI 추상화(3사), 검색(구글+네이버), 이미지(스톡+생성), 글/댓글 파이프라인, 자기검열·인젝션 방어, 오케스트레이터·안전장치, 관리자 대시보드, 일일 리포트·텔레그램 푸시.
- 설계 문서의 7인+관리자 페르소나·주제 풀·리듬을 **최초 시드**로 적재.

### Out of Scope (이번 제외)
- 공지사항 자동 작성(운영자가 직접 씀 — 설계 문서 명시).
- 봇이 사람 사용자 DM/외부 SNS로 능동 접촉하는 기능.
- 다국어 봇(한국어 전용).
- 카톡/슬랙 푸시(텔레그램 우선, 추후 확장 여지만 남김).

## 8. 운영 정책 (제품 차원 결정)

- **초기 1~2주 반자동 관찰**: 완전 자동이라도 처음엔 지켜보며 말투·주제 튜닝 후 풀어준다(권고).
- **점진적 감량**: 진짜 사용자가 늘면 봇 활동량을 단계적으로 줄인다(관리자에서 빈도 하향).
- **투명성 내부 기준**: 봇 계정은 `is_bot`로 내부 식별 가능. 외부 라벨 노출 여부는 운영자 정책 결정(기본: 비노출, 시딩 목적).
