## 1. 두 개념 분리 (중요)

같은 점수(`작당력` — 사용자가 활동으로 쌓는 기여 점수)를 쓰지만, 집계 방식이 다른 **두 층**이 존재한다.

| 구분 | 기준 | 성격 |
|---|---|---|
| **등급**(`grade`, 영구 계급) | **누적** 작당력 | 평생 쌓임·하락 없음. 이름 확정: 새내기 → 작당원 → 실전러 → 고수 → 마스터 (`UX-DR-A8`) |
| **작당 랭킹**(`ranking`, 주기 리더보드) | **기간 내 증가한** 작당력 | 매주/매달 리셋·경쟁형. 신규 유저도 상위 진입 가능 |

랭킹은 "누적 총점이 높은 사람"이 아니라 **"이번 주/이번 달에 가장 많이 기여한 사람"** 을 보여준다. (커뮤니티 초기 활성화 목적)

---

## 2. 작당력 적립표 (점수표)

핵심 원칙: **작성 행위보다 "받은 반응"(좋아요·다운로드)에 가중치** 를 둬, 글 많이 쓴 사람이 아니라 **남에게 도움 준 사람**이 위로 가게 한다. (스팸 양치기 방지)

| 활동 | 작당력 | 비고 |
|---|---|---|
| 글 작성(가이드/팁/사례 등) | +5 | 일일 상한(하루 인정 최대 적립) 적용 |
| 묻고답하기 질문 작성 | +2 | |
| 답변 작성 | +5 | |
| 실전자료 등록(ClamAV 스캔 통과·공개 시) | +15 | 가장 가치 높은 기여 |
| 내 글·답변이 받은 좋아요 | 개당 +2 | 자가 좋아요(본인 글 좋아요) 무효 |
| 내 자료가 받은 다운로드 | 개당 +1 | 중복 유저 제외, 일일 상한 |
| 내 자료가 받은 평점(4점 이상) | +3 | |
| 내 글·자료가 받은 북마크 | 개당 +1 | |
| 가입·프로필 완성 | +10 | 1회성 |

> 수치(가중치·상한)는 구현 Story에서 운영 데이터에 맞게 조정 가능. 비율(반응 > 작성)과 항목 구성은 확정.

### 어뷰징 방지 (`FR-9.1`, `packages/core`)
- 자가 추천(자가 좋아요·자가 다운로드) 무효
- **삭제·숨김 시 해당 활동 작당력 회수**
- 활동별 일일 상한
- 반복 등록 차단 (`@fastify/rate-limit` + core 규칙)
- 도움된 답변(`FR-3.7`)은 **작당력과 미연결** (정답 강요 방지)

## 3. 작당 랭킹 산정

- **집계 대상**: `points_ledger`(포인트 원장 테이블)에서 **해당 기간 안에 적립된 작당력만** 합산 (누적 총량 아님)
- **기간**: `작당 랭킹 (이번 주)`(월~일) / `작당 랭킹 (이번 달)` 2종
- **노출**: 메인 페이지 위젯 TOP 5 + 별도 `작당 랭킹` 페이지 TOP 50
- **동점 처리**: 같은 작당력이면 **먼저 달성한 사람** 우선
- **산정 시점**: 무거운 작업 → `ranking` 큐(BullMQ 백그라운드 큐)에서 worker가 주기 계산(멱등). 실시간 계산 금지.
- **캐싱**: 계산 결과는 Redis 캐시(`AR-17` 인기·랭킹 버퍼)
