# AI작당 어드민 — EXPERIENCE.md (어드민 콘솔 경험 spine)

> 이 문서는 어드민이 **어떻게 동작하는가**(IA·보이스·행동·상태·인터랙션·접근성·권한·여정)를 소유한다.
> **어떻게 보이는가**는 `DESIGN.md`. 토큰은 `{path.to.token}` 문법으로 교차참조한다.
> **충돌 시 이 spine과 DESIGN.md가 목업·import·코드보다 이긴다.**

## Foundation

**데스크톱 우선 운영 콘솔.** 사이드바 + 상단바를 갖는 `AdminShell` 레이아웃 위에서 테이블·필터·드로어·모달 중심의 백오피스 UI를 운영한다. 별도 서브도메인(`admin`)·별도 포트(3004)로 유저 사이트(3003)·API(4003)와 독립 실행한다.

**관리자 신원은 유저와 완전 분리(ADR-0003).** 별도 계정 테이블·별도 세션 쿠키(`aj_admin_session`)·별도 인증 인스턴스를 쓰며, **유저 사이트 가입/로그인은 관리자에 전혀 영향이 없다.** 관리자는 이메일+비밀번호(소셜 없음) + 이름·연락처를 받고, **가입 후 최고관리자 승인(승인 대기→정상)을 받아야 로그인**된다. 최초 최고관리자는 시드로 생성.

UI 시스템은 **`@ai-jakdang/admin-design-system`** — 프레임워크 비종속(순수 CSS + 바닐라 JS) 패키지로, 유저 사이트와 **토큰·색·타이포·간격·라운드·그림자·CSS·컴포넌트·레이아웃·아이콘을 일체 공유하지 않는다.** 어드민 앱은 유저 사이트의 `styles/* components/* features/*`를 import하지 않는다. 공유받는 것은 **비시각 패키지뿐** — `@ai-jakdang/contracts`(API 타입·Zod 스키마), `@ai-jakdang/auth`(권한 타입·`canAccessAdmin` 등 권한 검사 규칙). 진입점은 `initAdminUI()`, 아이콘은 `remixicon`.

핵심 제약 두 가지가 경험 전체를 지배한다:

1. **사후(事後) 운영 도구다.** AI작당은 비회원 읽기 개방 + 회원 즉시 등록(검수 없음) 구조다. 어드민은 **사전 승인자가 아니라 사후 개입자** — UX는 "검토 대기열 승인"이 아니라 "문제 발생 후 빠른 처리"에 최적화한다.
2. **권한 검사는 서버에서.** 화면의 권한 분기·메뉴 숨김은 UX 편의일 뿐, 실제 통제는 API(`/api/v1/admin/*`)가 한다. 클라이언트 권한값만 믿고 어드민 기능을 허용하지 않는다(NFR-2).

폼팩터는 **데스크톱 브라우저 콘솔**이 1순위. `980px`에서 사이드바 off-canvas, `760px`에서 1열 적층으로 **degrade**만 하며 모바일 전용 최적화는 비요구 `[ASSUMPTION]`. 단일 라이트 테마, 단일 언어(한국어), i18n 미적용 `[ASSUMPTION]`.

## Information Architecture

관리자 메뉴는 **14개 1차 메뉴**(아키텍처 단계에서 14. 문의 관리 추가) + 인증 라우트(`/login`) + 진입 리다이렉트(`/` → `/dashboard`). 좌측 사이드바(`AdminShell`)에 nav 그룹으로 묶어 배치한다.

```text
관리자 콘솔 (admin 서브도메인 / :3004)
├── /login                  로그인 (관리자 전용 신원 — status=active만 통과, 대기/정지/비활성은 사유 안내)
├── /signup                 관리자 가입 (이름·이메일·비밀번호·연락처 → status=pending 승인 대기)
├── / → /dashboard          루트 진입 시 대시보드로 리다이렉트 (로그인 + 승인된 계정만)

[그룹: Overview]
├── 1. 대시보드 (Dashboard)      /dashboard   [핵심] 요약 카드·운영 알림·콘텐츠 성과·마케팅 성과
├── 2. 접속 통계 (Analytics)                  [핵심] 기본 통계·기간 필터·유입 경로·검색 키워드·콘텐츠별 성과·다운로드 전환율

[그룹: Content]
├── 3. 게시글 관리 (Posts)                     [핵심] 목록·필터·행 처리(공지/고정/추천/메인노출/숨김/삭제/복구)
├── 4. 묻고답하기 관리 (Q&A)                   [핵심] 질문/답변 처리·상태 강제 변경
├── 5. 실전자료 관리 (Resources)              [핵심] 자료 처리·첨부파일 확인/부적절 삭제·후기·신고
├── 6. 댓글·후기 관리 (Comments)             [핵심] 댓글/답변/후기 통합·관련글 이동

[그룹: Operation]
├── 7. 신고 관리 (Reports)        ★통합 화면  [핵심] 신고 대상·사유·상태(접수/확인중/처리완료/반려)·처리
├── 14. 문의 관리 (Inquiries)    신규        [핵심] 1:1 문의 목록·상태(접수/처리중/완료)·답변 작성. 외주 문의 유입 대응
│   #   (공지 관리는 3.게시글 관리의 공지 게시판으로 처리하거나 별도 공지 관리 화면)

[그룹: Engagement]
├── 8. 회원 관리 (Members)                    [핵심] 목록·상태(정상/이용제한/탈퇴)·권한·포인트±·등급·뱃지·이력
├── 9. 포인트 관리 (Points)                   [기본형] 지급/차감 규칙·수동 지급·차감·비정상 적립
├── 10. 등급 관리 (Grades)                    [기본형] 등급 목록·기준·권한·수동 변경·자동 업데이트
├── 11. 뱃지 관리 (Badges)                    [기본형] 뱃지 목록·지급 조건·수동 지급/회수

[그룹: Business]
├── 12. 광고 관리 (Ads)          수익화       [핵심·최고관리자 전용] 광고 유형/위치/PC·모바일/기간/코드·성과
├── 13. 사이트 설정 (Settings)   [최고관리자 전용] 기본/콘텐츠/파일/신고 설정
```

**개발 우선순위(IA 깊이):**
- **핵심(깊게):** 대시보드, 접속 통계, 게시글, 묻고답하기, 실전자료, 댓글·후기, 신고, **문의(신규)**, 회원, 광고.
- **기본형(얕게 시작):** 포인트, 등급, 뱃지.
- **제외/후순위:** 포인트 교환소, 유료 자료 정산, 결제, 뱃지 미션, 복잡 랭킹, 자동 파일 보안검사, AI 자동검수, 실시간 알림, 쪽지/채팅 관리, 운영자 활동 로그 분석, A/B 테스트·캠페인 자동화.

**내비게이션** — 진입은 로그인 → `/dashboard`, 루트는 항상 대시보드 리다이렉트. nav active는 `{colors.primary-50}` 배경·`{colors.primary-700}` 글자. 미처리 신고 등은 nav 배지(`{components.nav-badge}`, danger pill)로 노출. **크로스 링크**: 댓글·후기 → "관련 게시글로 이동", 신고 → "신고 대상 보기"(대상 콘텐츠로 이동). 권한 없는 메뉴는 **숨김**(아래 Permissions 참조).

## Voice and Tone

마이크로카피. 브랜드 보이스·미감은 `DESIGN.md.Brand & Style`. 어드민 톤은 **실무적·간결·결정 유도형** — 대시보드는 "예쁜 차트"가 아니라 "오늘 무엇을 봐야 하는지"를 알린다 `[ASSUMPTION]`. 파괴적 액션 카피는 **판단을 환기하는 톤**(신고가 들어와도 무조건 지우지 않는다는 원칙과 일관).

**도메인 어휘(운영 라벨) — 일관 사용:**
- 콘텐츠 상태: `공개 / 숨김 / 삭제`
- 회원 상태: `정상 / 이용 제한 / 탈퇴`
- 신고 상태: `접수 / 확인중 / 처리완료 / 반려`
- Q&A 상태: `답변대기 / 답변있음 / 해결됨`
- 행동 동사: `숨김 처리 / 삭제 처리 / 복구 처리 / 반려 / 이용 제한 / 메모 작성 / 강제 변경 / 회수 / 수동 지급·차감`
- 등급명(브랜드 톤): `새내기 / 작당원 / 실전러 / 고수 / 마스터 / 운영자`

| Do | Don't |
|---|---|
| "미처리 신고 3건 — 확인이 필요합니다." | "신고 3" |
| "이 콘텐츠를 삭제합니다. 사유를 입력해 주세요." | "정말 삭제하시겠습니까?" (사유 없이) |
| "숨김 처리했습니다. [되돌리기]" | "처리 성공 ✓" |
| "조건에 맞는 신고가 없습니다." | "결과 없음" |
| 상태 라벨을 정의 어휘 그대로 일관 사용 | 화면마다 다른 표현(가림/비공개/내림 혼용) |

## Permissions & Destructive Actions (제품 특화 — 확정 결정)

> 결정 로그의 사용자 확정 결정 1·2·3을 통합한다. 권한 enforcement의 **최종 통제는 API 서버**이며, 아래는 화면 경험 규정이다.

### 역할 매트릭스 (3단계)

| 권한 | 어드민 접근 | 가능 범위 | 상한선 |
|---|---|---|---|
| **최고관리자 (super admin)** | 전체 | 모든 어드민 기능 · **영구삭제** · **사이트 설정** · **광고 관리** · **회원 권한 변경** · 운영자 권한 부여 | 없음 |
| **운영자 (staff/moderator)** | 일부 | 게시글·댓글·신고·실전자료·Q&A 관리, 회원 일부 조회, **숨김(soft)까지** | 영구삭제·사이트설정·광고·회원 권한변경 **불가** |
| **일반회원** | 불가 | 어드민 페이지 전면 차단 | 접근 차단 |

- **운영자 상한선 = 숨김(soft).** 운영자는 콘텐츠/자료/댓글을 숨김·플래그 토글·신고 반려·메모·이용제한까지 할 수 있으나, **영구삭제(hard-delete)·사이트 설정·광고·회원 권한 변경은 최고관리자 전용**이다. (기획서 §17의 "삭제=최고관리자" vs 운영자 메뉴의 삭제 기능 모순을 이 결정으로 해소.)
- **권한 없는 메뉴/액션은 숨김(hidden) + 접근 시 거부.** 운영자에게 광고·사이트 설정 메뉴는 사이드바에서 보이지 않으며, 직접 URL 접근 시 권한 거부 상태(아래 State Patterns)로 처리된다.
- 권한 검사 규칙은 비시각 패키지 `@ai-jakdang/auth`(`canAccessAdmin` 등)에 있고, **API가 최종 통제**한다.
- **관리자는 유저와 별개의 신원/계정/세션이다(ADR-0003).** 위 매트릭스의 "일반회원 차단"은 *사이트 회원 계정으로는 어드민에 들어올 수 없다*는 의미이며, 관리자 계정은 별도 가입(`/signup`) + **최고관리자 승인** 후에만 활성화된다. 운영자/최고관리자 역할(`AdminRole`)도 유저 역할과 분리.

### 위험도별 파괴적 확인 UX (risk-tiered)

신고 처리 5단계 가드레일(대상 확인 → 사유 확인 → 위반 판단 → 숨김/삭제/반려 → 제재)과 일관되게 **위험도에 따라 확인 강도를 차등**한다.

| 위험도 | 액션 | 확인 패턴 |
|---|---|---|
| **되돌릴 수 있음** | 숨김(soft), 플래그 토글(공지/고정/추천/메인노출), 신고 상태 변경 | **즉시 실행 + 토스트(undo 노출).** 모달 없음 |
| **위험·되돌리기 어려움** | **삭제(hard) · 이용 제한(밴) · 포인트 차감 · 뱃지 회수 · 권한 변경** | **확인 모달 + 사유 메모 입력 필수.** 사유 없이는 실행 불가 |

- 즉시 실행 액션은 `{components.toast}`로 결과를 알리고 **되돌리기 버튼**을 제공한다.
- 위험 액션은 `{components.modal}`로 멈춰 세우고 **사유 메모**를 받는다 — 처리 이력·신뢰 추적용.

### 삭제 보존 (soft-delete + 자동 hard-delete)

- 콘텐츠/자료/댓글 삭제는 **soft-delete 기본**(상태 `삭제`로 전이, 목록에서 제외).
- **보존기간(예: 30일) 내 복구(restore) 가능** — "휴지통"에서 되살린다.
- 보존기간 경과 시 **자동 영구삭제(hard-delete)** — "휴지통 자동 비움".
- 운영자는 soft-delete(숨김 상한)까지만, **자동 hard-delete 시점·수동 영구삭제는 최고관리자/시스템** 영역.
- 보존기간 값은 사이트 설정(신고/콘텐츠 설정) 노출 후보 `[NOTE FOR UX]`.

## Component Patterns

행동 규칙. 시각 사양은 `DESIGN.md.Components`.

| 컴포넌트 | 사용처 | 행동 규칙 |
|---|---|---|
| 데이터 테이블 | 거의 모든 1차 메뉴(게시글/Q&A/자료/댓글/신고/회원) | 표준 컬럼군: 제목·작성자·작성일 + 활동지표(조회/댓글/좋아요/다운로드/신고) + 상태 + 플래그. 행 클릭/보기 = 상세(드로어 또는 상세 화면). 수치 셀은 `{colors.gray-700}` + tabular-nums |
| 필터 패널 | 모든 목록 메뉴 | 메뉴별 전용 필터셋(게시판별/상태/공지·추천·메인노출/기간/작성자/신고여부/자료유형·환경·난이도/Q&A 상태/댓글 유형). 적용 시 활성 필터칩으로 노출, URL 쿼리 반영 `[ASSUMPTION]` |
| 행 액션 메뉴 | 테이블 행 | 보기·수정·숨김·삭제·복구·플래그 토글. danger 항목(`{components.table-row-action.danger-item-color}`)은 위험도별 확인 패턴 따름 |
| 벌크(일괄) 액션 | 테이블 툴바 | 다중 선택(체크박스) 후 선택 정보 표시 + 일괄 버튼(일괄 숨김/삭제 등). 선택 행은 `{colors.primary-50}` 배경. 일괄 파괴 액션도 위험도별 확인 적용 |
| 라인 탭 | 목록 내 상태·하위 영역 전환 | active는 `{colors.primary-700}` + 언더라인. 전환은 목록 필터/뷰만 바꾸고 화면 이동 아님 |
| 커스텀 셀렉트 | 필터·폼 | 클릭 열기, 방향키 이동, Enter/Space 선택, Esc·바깥클릭 닫기. 네이티브 값 동기화. 네이티브 select 노출 금지 |
| 폼 (모달/드로어/페이지) | 설정·수정·규칙 편집(사이트 설정/광고/포인트·등급·뱃지 규칙) | RHF + Zod 검증. 저장중/완료 토스트. 신고·회원 상세는 "메모 작성"·"상태 변경" 인라인 편집 |
| 상세 드로어 | 행 상세 보기 | 우측 슬라이드, detail-list(라벨 + 값). 상세 → 처리 액션 인라인 노출 |
| 토스트 | 처리 결과 | 즉시 실행 액션 결과 + undo. success/error 구분. 3~5초 |
| 대시보드 위젯 | 대시보드 | KPI 요약 카드 + 운영 알림 리스트 + 랭킹 리스트(인기/전환 상위) + 차트(Recharts 래퍼). "나쁜 신호"(미처리 신고·답변대기·저품질 자료) 우선 노출 |

**운영자가 직접 하지 않는 행위(가드레일):**
- Q&A: 질문자의 "도움된 답변" 대신 지정 금지(질문자 권한 유지).
- 댓글: 회원 댓글 **내용 직접 수정 금지**(숨김/삭제 중심) — 신뢰 문제.
- 실전자료: 검수됨/공식/안전성 보증/자동 보안검사 **미제공**(사후관리자이지 검수자가 아님).

| 플래그 토글(flag toggle) | 공지/고정/추천/메인노출, 자동지급·자동등급 ON/OFF | **즉시 실행 + 토스트(undo)** — 되돌릴 수 있는 액션 |
| 파괴적 확인(confirm-destructive) | 삭제/이용제한/포인트차감/뱃지회수/권한변경 | **확인 모달 + 사유 메모 입력 필수** — 되돌리기 어려운 액션 |

위험도별 규칙 전체는 `## Permissions & Destructive Actions`. 핵심: **되돌릴 수 있으면 즉시+undo, 되돌리기 어려우면 모달+사유.**

## Accessibility Floor

행동 규칙. 시각 대비는 `DESIGN.md`. 소스에 접근성 명시 요구 없음 → 데스크톱 콘솔 최소선 `[ASSUMPTION]`.

- **키보드 내비게이션** — 테이블 행/필터/모달 포커스 이동. 모달·드로어는 **포커스 트랩** + 배경 스크롤 잠금. Esc는 최상위 오버레이를 닫는다. Tab 순서 = 읽기 순서.
- **상태를 색만으로 구분하지 않음** — 7색 시맨틱 배지에 텍스트 라벨 병기(`badge`/`alert` 컴포넌트). 색각 이상·흑백 출력 대응.
- **파괴적 액션** — 명확한 라벨 + 확인 + 포커스 이동(스크린리더 대비).
- **차트** — 표/수치 대체 제공(차트 데이터 alternatives).
- **커스텀 셀렉트** — `aria-haspopup`/`aria-expanded`/`role=listbox`/`role=option`. icon-only 버튼 `aria-label` 필수. 포커스 링(`{shadow.focus-ring}`) 제거 금지.
- **`prefers-reduced-motion`** 존중(모든 애니메이션 0.01ms로 무효화).
- WCAG 목표 등급 `[ASSUMPTION]` — 미정(`[NOTE FOR UX]`).

## Key Flows

명명된 주인공 여정. climax beat는 **굵게**. 권한 경계는 위 매트릭스를 따른다.

### Flow A — 신고 처리 (지훈, 운영자, 매일 아침 / climax: "삭제할까, 살릴까")

1. 지훈은 대시보드 `운영 알림 — 미처리 신고 N건`을 확인한다.
2. **신고 관리**(통합 목록) 진입 → 신고 행 클릭 → 신고 대상 콘텐츠 보기 + 신고 사유·신고자·작성자 확인.
3. 대상 콘텐츠를 직접 열어 **실제 위반 여부를 판단**한다.
4. **climax:** 신고가 들어왔다고 무조건 지우지 않는다 — "정상 글을 잘못 지우면 신뢰가 깨진다". 위반이면 **숨김(즉시+undo)**, 아니면 **반려**, 악질이면 작성자 **이용 제한(모달+사유)** + 관리자 메모 → 상태를 `처리완료/반려`로 변경.
5. 권한: 운영자 가능 영역. **영구삭제는 하지 않고 숨김까지** — hard-delete는 최고관리자.

### Flow B — 좋은 콘텐츠 메인 노출 (은영, 최고관리자 / climax: "이 자료를 메인에 올린다")

1. **접속 통계**에서 다운로드 전환율 높은 자료 발견(예: 조회 300/다운 80 = 26.6%).
2. **게시글/실전자료 관리**로 이동 → 해당 자료 행에서 **메인 노출 설정** + 추천 지정 + (필요 시) 상단 고정.
3. **climax:** "조회 1,000에 다운 20(2%)보다, 조회 300에 다운 80인 자료가 더 강하다" — 은영은 숫자로 강한 자료를 골라 메인에 띄운다. 플래그 토글은 **즉시 실행 + 토스트(undo)**. 운영 = 마케팅 판단 도구.

### Flow C — 위험 첨부파일 사후 관리 (지훈, 운영자 / climax: "악성 의심 파일을 내린다")

1. 신고(`악성 파일 의심`) 또는 실전자료 신고 내역 → **실전자료 관리** 상세.
2. **첨부파일 확인** → 부적절 첨부파일 삭제 또는 자료 숨김.
3. **climax:** AI작당은 "누구나 등록" 정책 + "운영자는 검수자가 아니라 사후관리자"라는 선을 지킨다. 지훈은 안전성을 **보증하지 않되**, 위험 자료는 빠르게 내린다(자동 보안검사 없음 = 사람이 판단). 영구삭제가 필요하면 최고관리자에게 넘긴다.

### Flow D — 회원 활성화 보상 (은영/지훈 / climax: "수동으로 포인트·뱃지를 준다")

1. **회원 관리** 상세 → 활동 내역(글/답변/도움된답변/자료) 확인.
2. **포인트 수동 지급** 또는 **뱃지 수동 지급** / 필요 시 **등급 수동 변경**.
3. **climax:** 자동 규칙으로 못 잡는 "기여한 회원"을 운영자가 직접 알아보고 보상해 참여를 끌어올린다. 단 포인트 차감·뱃지 회수는 **모달 + 사유 메모**(되돌리기 어려운 액션). 포인트는 현금성 보상과 비연결 — 활동 지표일 뿐.

### Flow E — 위반 회원 제재 (은영, 최고관리자 / climax: "이용 제한을 건다")

1. 신고 누적/심각 위반 확인 → **회원 관리** 상세 → 신고 이력 보기.
2. **회원 상태 변경: 이용 제한**(+ 포인트 차감, 위반 콘텐츠 삭제) — 모달 + 사유 메모 필수.
3. **climax:** "신고 누적 후 위반 확정" 시점에만 제재 — 절차를 거친 제재. 밴은 되돌릴 수 있는 상태 변경(정상/이용제한/탈퇴)으로 관리. 회원 권한 변경·영구삭제는 **최고관리자 전용**.

> **Epic 12 확장 — 회원 직접 신고 & 누적 에스컬레이션 (2026-06-30)**
> - **회원 직접 신고(FR-8.6):** 유저는 콘텐츠뿐 아니라 회원 자체(프로필 부적절·사칭·도배·괴롭힘)를 신고할 수 있다. 이 신고는 **신고 관리** 통합 큐에 `대상 유형=회원`으로 들어오고, "신고 대상 보기"는 **회원 상세**로 연결된다(콘텐츠 숨김 대신 "회원 제재" 액션 노출).
> - **처리완료 누적 → 자동 경고 + 에스컬레이션(FR-8.7):** 관리자가 **처리완료(resolved)** 한 신고만 작성자에게 누적된다(접수 수 아님 — 브리게이딩 방어). 임계치(사이트 설정 `report_escalation_threshold`, 기본 5) 도달 시 **자동 경고 1건 + "검토 요망" 에스컬레이션**(대시보드·nav danger pill). **자동 일시정지/영구정지는 없다** — 정지 이상은 은영이 Flow E 2~3단계로 수동 판단. 자동화는 기본 OFF(`report_auto_warning_enabled`).
> - **제재 통보(FR-8.8):** 경고·정지·영구밴(자동 경고 포함) 부여 시 당사자에게 인앱 알림(사유·종료일).
> - 회원 상세 "피신고 이력" 섹션에 처리완료 신고수·목록 + 임계치 근접 시 danger 배지를 노출해 "절차를 거친 제재" 원칙을 데이터로 뒷받침한다.

### Flow F — 광고 수익 위치 운영 (최고관리자 전용 / climax: "신뢰를 해치지 않는 선에서 단다")

1. **광고 관리**(운영자에게는 메뉴 숨김) → 광고 등록(유형/위치/PC·모바일/노출기간/코드).
2. 성과(노출/클릭/CTR/기간·위치·기기별) 모니터링.
3. **climax:** "다운로드 버튼 근처에 광고를 과하게 넣으면 신뢰가 떨어진다" — 수익과 신뢰 사이에서 위치를 절제. 광고 영역은 **최고관리자 전용 권한 경계** — 운영자는 접근 불가(메뉴 숨김 + 거부).

## Open Questions / [NOTE FOR UX]

다음은 spine을 막지 않는 미해결 항목이다. 메인 에이전트가 화면 설계 시 결정·보강한다.

1. **정렬 컬럼 구체 목록** — 목록별 정렬 가능 컬럼(조회/다운로드/신고/작성일 등)과 기본 정렬, 페이지네이션 vs 무한스크롤 확정.
2. **에러 상태 표준 UX** — API 오류 표시 패턴(인라인/토스트/전체화면) 미정의.
3. **WCAG 목표 등급** — 접근성 준수 수준 미정 `[ASSUMPTION]`.
4. **로그인/인증 상세** — `/login` 화면, 인증 방식(Better Auth 이메일 기반 추정), 세션·2FA 여부.
5. **저니 없는 서피스** — 접속 통계 심화(키워드/유입 분석), 사이트 설정(SEO·파일정책·신고설정), 등급/뱃지 규칙 편집 화면의 운영자 여정(누가 언제 왜) 미정의.
6. **자동 숨김 임계치 값** — 신고 누적 자동 숨김의 ON/OFF 기본값·임계치. 사이트 설정 노출 후보.
7. **보존기간 값** — soft-delete 자동 hard-delete까지 기간(예: 30일)을 사이트 설정으로 노출할지.
8. **회원 "일부 조회" 범위** — 운영자가 볼 수 있는 회원 PII(이메일 등) 범위.
