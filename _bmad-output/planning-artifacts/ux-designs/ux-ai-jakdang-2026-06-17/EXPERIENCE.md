# AI작당 — EXPERIENCE.md (유저 사이트 경험 spine)

> 이 문서는 **어떻게 동작하는가**(IA·행동·상태·인터랙션·접근성·여정)를 소유한다.
> **어떻게 보이는가**는 `DESIGN.md`. 토큰은 `{path.to.token}` 문법으로 교차참조한다.
> 충돌 시 이 spine과 DESIGN.md가 목업·와이어프레임·코드보다 이긴다.

## Foundation

**반응형 웹**(데스크톱/태블릿/모바일 단일 코드베이스). Next.js App Router `apps/web`, **자체 제작 디자인 시스템**(외부 UI/CSS 프레임워크 미사용), CSS Modules + CSS 변수 토큰, Pretendard, Remix Icon. `DESIGN.md`가 비주얼 아이덴티티 레퍼런스이고, 이 spine이 경험을 규정한다.

핵심 제약 두 가지가 경험 전체를 지배한다:

1. **SSR / SEO가 #1 과제(NFR-1).** 공개 페이지(목록·상세·태그·프로필)는 서버 렌더링으로 검색엔진·AI 검색이 본문을 즉시 파싱할 수 있어야 한다. 클라이언트 전용 렌더링으로 본문을 숨기지 않는다.
2. **비회원 읽기 개방 · 행동 게이팅(전환 정책 확정).** 목록·상세·검색은 비회원에게 완전 개방(SEO·유입 보호). 다운로드·작성·등록·반응(좋아요·북마크·댓글·평점·후기·쪽지·신고)은 로그인 필요 → 로그인 유도. 자세히는 `## 권한 & 게이팅`.

향후 React Native 앱과 로직 공유를 대비하나(NFR-7) Phase 1은 웹만. 어드민(`apps/admin`)은 별도 앱·별도 디자인 시스템으로 이 spine 범위 밖.

## Voice and Tone

마이크로카피. 브랜드 보이스·미감은 `DESIGN.md.Brand & Style`. **기본 말투는 정중한 존댓말**, 성격은 "차분한 실전 동료" — 과장·이모지 남발 없이 사실과 다음 행동을 명확히.

| Do | Don't |
|---|---|
| "검색어를 입력하세요" | "무엇이든 찾아보세요! 🔍" |
| "아직 답변이 없어요. 첫 답변을 남겨보세요." | "답변 0개" |
| "로그인하고 이 자료를 받아가세요. 가입은 30초면 됩니다." | "다운로드하려면 로그인하십시오." |
| "해결됨으로 표시했습니다." | "상태 변경 성공 ✓" |
| "조건에 맞는 글이 없어요. 태그를 바꿔보세요." | "결과 없음" |
| 모두에게 같은 정중한 존댓말 | 화면·대상마다 말투를 바꾸기 |

게이팅(로그인 유도) 톤 = **가치 강조 + 부드러운 권유**: 막는 게 아니라 이득을 말하며 권한다. 압박·과장 금지.

## 권한 & 게이팅 (제품 특화)

PRD 5-1 역할표 기준. 읽기는 막지 않고, "행동"에서만 로그인을 요구한다.

| 행동 | 비회원 | 회원 | 게이팅 처리 |
|---|---|---|---|
| 목록·상세·검색 열람 | ✅ | ✅ | 게이팅 없음(SSR로 본문 노출) |
| 자료 다운로드 | ❌ | ✅ | 로그인 유도 모달(가치 강조). 다운로드 버튼은 보이되 클릭 시 모달 |
| 글·질문·답변·자료 작성/등록 | ❌ | ✅ | [글쓰기]/[등록] 클릭 시 로그인 모달. "로그인 후 등록" 라벨 |
| 반응(좋아요·북마크·댓글·평점·후기) | ❌ | ✅ | 인라인 액션 클릭 시 로그인 모달 |
| 쪽지·신고 | ❌ | ✅ | 로그인 모달 |

**원칙:** 게이팅 모달은 막지 말고 권한다. 비회원에게도 행동 진입점(버튼)을 보여 주어 "여기서 뭘 할 수 있는지"를 알린 뒤, 클릭 순간 가치를 곁들여 로그인으로 잇는다(전환 레버). 로그인 후 **원래 하려던 행동으로 복귀**한다(다운로드 시도 → 로그인 → 자동 다운로드 시작).

## SEO & 구조화 (제품 특화 — #1 과제)

UX가 SEO를 직접 떠받친다. 디자인·행동 결정이 검색 노출을 깨지 않도록 한다.

- **본문은 SSR로 즉시 파싱 가능**하게. 무한 스크롤 대신 **페이지네이션**(색인 안정성). 탭/필터 전환이 본문을 클라이언트에서만 바꿔 끼우지 않도록(딥링크 가능 URL 유지).
- 페이지당 **H1 1개**, 명확한 H2/H3, **breadcrumb**(JSON-LD BreadcrumbList).
- 페이지마다 고유 title·meta description **자동 생성**(회원 입력 없이, 운영자만 보정) — 작성 화면에 SEO 입력 비노출.
- **태그 페이지(`/tags/{tag}`)는 1급 SEO 랜딩.** 태그는 키워드 진입점이자 탐색 도구.
- 외부 링크는 **링크 OG 카드**(제목·도메인·설명·이미지)로(FR-11.7). 이미지 alt 필수.
- 저품질·빈 페이지는 noindex(검색 평판 보호, FR-11.9).
- 본문 렌더링은 **XSS 차단**(HTML/script 실행 금지) — 줄바꿈·들여쓰기·코드블록은 보존.

행동 규칙. 시각 사양은 `DESIGN.md.Components`.

| 토스트 | 저장·복사·다운로드 시작·오류 | 짧은 피드백 3~5초. 오류는 길게/수동. 액션은 즉시행동만(되돌리기 등) |
| 검색 자동완성 | 헤더 검색 | 최근검색어·추천태그·인기검색어 구분. 방향키·Enter·Esc |
| 페이지네이션 | 모든 목록 | 무한 스크롤 미사용. `aria-current=page`. 모바일 축약형(현재/전체 + 이전·다음) |

## Interaction Primitives

마우스·터치·키보드 동등 접근. 파워 유저(개발자) 비중이 높으나, 정보 탐색·전환이 우선이라 키보드는 *동등 접근 보장* 수준(전용 단축키 체계는 두지 않음).

- **모든 인터랙션 요소** = hover/active/focus/disabled 상태 보유. hover에만 의존하는 기능 금지(터치 대응).
- **오버레이**(드롭다운·팝오버·모달·드로어·자동완성) = Esc·바깥클릭 닫기, 방향키 이동, Enter/Space 선택. 모달·드로어는 포커스 트랩 + 배경 스크롤 잠금.
- **코드블록** = 복사 버튼, 가로 스크롤, 들여쓰기·특수문자 보존(FR-2.6).
- **필터·정렬·탭** = URL에 반영(뒤로가기·공유·딥링크·SEO).
- **터치 영역** 최소 36~44px.
- **금지**: 무한 스크롤(페이지네이션만), hover-only 어포던스(모바일), 모달 2단 스택, 색상 단독 상태 전달.

## Accessibility Floor

행동 규칙. 시각 대비는 `DESIGN.md`.

- 버튼=`<button>`, 링크 이동=`<a>`. icon-only 버튼 `aria-label` 필수.
- 커스텀 Select=`aria-haspopup`/`aria-expanded`/`role=listbox`/`role=option`. 탭=`role=tablist`/`aria-selected`. 아코디언·드롭다운 트리거=`aria-expanded`. 현재 페이지=`aria-current=page`.
- 키보드 포커스 표시 제거 금지(`{shadow.focus-ring}`). Tab 순서 = 읽기 순서. Esc는 최상위 오버레이를 닫는다.
- 색상만으로 상태 전달 금지 — 의미색에 아이콘·텍스트 동반.
- disabled = 클릭 차단 + 시각 약화 동시 적용. 이미지 alt·명확한 헤딩 구조(SEO와 공유, NFR-5).
- `prefers-reduced-motion` 존중(`base/accessibility.css`).

## Inspiration & Anti-patterns

- **차용(커뮤니티 관례):** 텍스트형 게시판 리스트·정렬 탭·태그 기반 탐색 — 한국 실전 커뮤니티의 빠른 정보 탐색 패턴.
- **거부 — 게시판 과도 분할 / 툴별 게시판 초기 생성:** 태그로 처리(FR-6.4). 질문 게시판도 단일 통합(분산 금지).
- **거부 — 채택/정답/내공/마감 메커니즘:** '도움된 답변'은 약한 표시일 뿐, 정답·보상·마감을 만들지 않는다(FR-3.7). 답변 부담을 낮춘다.
- **거부 — 무거운 게이미피케이션 전면화:** 포인트는 뒷단 회계(노출 최소), 보이는 건 등급·뱃지·랭킹뿐. **핵심 활동에 포인트/등급 제한을 두지 않는다**(FR-9). 글쓰기 등은 항상 가능.
- **거부 — 목록 썸네일 혼재 / 복잡 입력폼 / 큰 히어로 비주얼:** 정보 밀도·등록 단순성·탐색 속도 우선(가드레일 5-3).
- **거부 — MCP/Skill 안전성을 사이트가 보증하는 듯한 표현:** 주의사항은 권장하되 사이트가 검수·보증하지 않는다(검수·공식 배지 미사용).

## Key Flows

명명된 주인공 여정. climax beat는 **굵게**.

### Flow 1 — 검색 유입 후 가입 (지훈, 외주 개발자, 평일 밤 막힌 순간)

1. 지훈은 "cursor php 구조 잘못 이해"를 검색한다.
2. AI작당 글이 검색 상단에 노출된다(SSR·정확한 title·메타).
3. 비회원 그대로 본문을 읽고 코드블록 [복사] 버튼으로 코드를 가져간다(읽기·복사는 게이팅 없음).
4. 본문 하단 관련 태그 `#cursor`를 눌러 `/tags/cursor`에서 더 탐색한다.
5. **climax:** 검색 한 번으로 들어온 비회원이 "막힌 문제의 답 + 관련 글 더미"를 즉시 손에 넣는다 — 막히는 게 없으니 신뢰가 생겨 가입하고, 다음 막힘에서 묻고답하기에 질문한다. 유입→가치체감→전환이 한 흐름.

실패: 검색은 됐지만 본문이 빈약 → noindex 정책으로 빈 유입 방지. 가입 모달에서 이탈 → 읽기는 유지되어 다음 방문에 재전환 가능.

### Flow 2 — 질문하고 답을 얻기 (수민, 자동화 입문자)

1. 수민은 "n8n으로 Gmail 문의 자동분류 가능?"을 묻고답하기에 게시한다(상태=답변대기, warning 배지).
2. 다른 회원이 답변(확장 댓글형, 코드블록 포함)을 남기자 상태가 답변있음으로 바뀐다.
3. 수민은 실제로 참고한 답변에 '도움된 답변'을 단다(1개, 부담 없는 표시, 보상 없음).
4. 문제를 해결한 뒤 질문자 버튼으로 직접 '해결됨'(success)으로 바꾼다.
5. **climax:** 채택·내공·마감 같은 무게 없이, 질문 하나가 "답변대기 → 답변있음 → 해결됨"으로 가볍게 닫힌다. 답변자는 받은 좋아요로 인정받고(포인트는 뒷단), 수민은 정답 압박 없이 문제를 푼다.

실패: 답변이 안 달림 → 빈 상태가 아니라 답변대기로 노출, 관련 태그로 더 많은 눈에 띔.

### Flow 3 — 실전자료 다운로드 (민지, 프리랜서)

1. 민지는 실전자료에서 "MCP" 유형 탭 + "Claude Code" 지원환경 필터를 건다(URL에 반영).
2. 카드에서 평점·다운로드 수로 신뢰를 가늠한다.
3. 상세에서 "이 자료는 무엇인가요"·사용법·주의사항을 확인한다.
4. [다운로드]를 누르자 비회원이라 로그인 유도 모달이 뜬다("로그인하고 받아가세요, 가입은 30초"). 로그인하면 **원래 하려던 다운로드가 자동 시작**된다(원행동 복귀).
5. **climax:** 다운로드 게이트가 벽이 아니라 부드러운 전환점이 된다 — 자료의 가치를 이미 본 상태에서 30초 가입으로 받아가고, 적용 후 후기 댓글·평점을 남겨 다음 사람의 신뢰 판단을 돕는다.

실패: 파일이 설명과 다름 → 후기·신고로 처리(별도 오류 제보 없음). 신고 누적 시 자동 숨김 → 운영자 검토.

### Flow 4 — 자료 기여 (현우, 회원)

1. 현우는 자기 Cursor Rules를 zip으로 [등록]한다.
2. 단일 폼 흐름(유형 선택 → 공통 정보 → 첨부 → 사용법/주의사항 → 태그 → 미리보기)을 따른다. 유형이 달라도 입력칸은 같고 안내 문구만 바뀐다.
3. 검수 없이 즉시 공개된다(검수·공식 배지 없음).
4. 다운로드·평점·후기가 쌓이고, 받은 다운로드·좋아요로 포인트가 적립돼 등급·뱃지(자료 기여자)가 올라간다.
5. **climax:** "내 설정을 올렸더니 즉시 공개되고 남들이 받아 간다"는 즉각적 기여 경험 — 복잡한 검수 장벽 없이 커뮤니티에 자산이 쌓이고, 명예(등급·뱃지)로 가볍게 보상받는다.

실패: 타인 자료 무단 등록 → 등록 시 저작권/배포 권한 동의 체크(FR-14.2). 부적절 자료 → 신고·자동 숨김·운영자 관리.
