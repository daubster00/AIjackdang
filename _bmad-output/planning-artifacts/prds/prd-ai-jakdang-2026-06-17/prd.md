> **상태**: Fast path 초안. `[ASSUMPTION]` 태그는 기획서에 명시되지 않아 작성자가 추론한 항목 — 검토 후 확정/수정 필요.

## 1. 개요 & 비전

**AI작당**은 바이브 코딩 · AI 업무 자동화 · AI 수익화를 **실제로 시도하는 사람들**이 경험·자료·막힌 질문을 함께 쌓아가는 실전형 AI 커뮤니티다.

단순 AI 뉴스/정보 게시판이 아니라, AI 도구로 개발·자동화·외주·템플릿 판매·수익화까지 연결하려는 사람들을 위한 공간이다. 흩어진 AI 정보를 한곳에서 빠르고 정확하게 얻고, 자기 환경에 맞춰 실전에 적용하게 하는 것이 핵심 가치다.

**한 줄 콘셉트**: "AI로 만들고, 자동화하고, 돈으로 연결하는 실전 커뮤니티"

**차별점**: AI 도구를 실제로 써먹는 사람들을 위한 실전형 커뮤니티. (바이브 코딩 / Claude Code·Cursor·Codex / MCP·Skills·Rules·프롬프트 / n8n·Make·Zapier 자동화 / 외주·템플릿 판매·자동화 구축 수익화 / 다운로드형 실전자료)

## 2. 목표 & 성공 지표

### 2-1. 비즈니스 목표
- **검색 유입 기반 성장**: 콘텐츠가 검색엔진·AI 검색에 정확히 노출되어 자연 유입을 유도한다. (본 제품의 #1 과제)
- **외주 문의 유입**: 가장 현실적인 1순위 수익 경로.
- 초기 콘텐츠 축적: 운영자 주도로 **최소 50개, 목표 100개** 글 확보 후 본격 홍보.

### 2-2. 성공 지표 (Success Metrics)
- 검색엔진 색인 페이지 수 / 검색 유입 세션 수
- 주요 키워드(claude code, cursor, n8n, mcp 등) 검색 노출·클릭 수
- 월간 활성 사용자(MAU), 신규 가입 수
- 콘텐츠 생산: 글/질문/실전자료 등록 수
- 실전자료 **다운로드 수**(자료 핵심 지표), 평점, 후기 수
- 외주 문의 발생 건수

### 2-3. 카운터 지표 (Counter-Metrics)
- 스팸/광고성 글·자료 비율, 신고 처리 대기 건수
- 검색 노출은 늘지만 체류·재방문이 따라오지 않는 "빈 유입"
- 저품질 자료(낮은 평점·미작동 후기) 비율
- `[ASSUMPTION]` 게이미피케이션(포인트/등급)이 어뷰징(반복 글·자가 추천)을 유발하는지

## 3. 대상 사용자

타깃은 **AI 활용에 관심 있는 전 연령대 사용자**이며, 실전 동기로 진입한다.

- **바이브 코더**: AI 코딩 도구(Claude Code, Cursor, Codex 등)로 개발을 더 빠르게 하려는 사람.
- **자동화 실무자**: n8n·Make·Zapier·Apps Script로 업무/사업 시간을 줄이려는 사람.
- **AI 수익화 추구자**: 외주·템플릿 판매·자동화 구축 대행으로 돈을 벌거나 비용을 줄이려는 사람.
- **정보 탐색자**: 막힌 문제 해결, 도구 선택, 방향 검토를 위해 검색으로 유입되는 사람(상당수가 비회원 → 검색 유입 → 가입 전환).
- **운영자**: 초기 핵심 콘텐츠를 직접 생산하고 커뮤니티를 관리.

사용자 역할 (**전환 정책 확정**: 읽기·검색은 개방, 모든 "행동"은 로그인 필요):
- **비회원**: 목록/상세 **열람·검색만** 가능. **다운로드·작성·등록·반응(좋아요·북마크·댓글·평점·후기·쪽지·신고) 불가 → 로그인 유도.** (읽기는 막지 않음 — SEO·유입 보호.)
- **일반 회원**: 글/질문/답변/실전자료 작성·등록, **자료 다운로드**, 본인 콘텐츠 수정·삭제, 댓글·좋아요·북마크·평점·후기·신고·쪽지·알림.
- **운영자(관리자)**: 사후 관리(공지·고정·추천·메인 노출·숨김·삭제·신고 처리·댓글 관리), 통계, 광고 관리, SEO 메타 수정.

## 4. 사용자 여정 (User Journeys)

**UJ-1 — 검색 유입 후 가입 (지훈, 외주 개발자)**
지훈은 "cursor php 구조 잘못 이해" 검색 → AI작당 글이 상단 노출 → 본문 읽고 코드블록 복사 → 관련 태그 페이지(`/tags/cursor`)에서 더 탐색 → 유용해서 가입 → 다음 막힘에서 묻고답하기에 질문.

**UJ-2 — 질문하고 답을 얻기 (수민, 자동화 입문자)**
수민은 "n8n으로 Gmail 문의 자동분류 가능?"을 묻고답하기에 게시(상태=답변대기) → 답변이 달리며 답변있음 → 실제로 참고한 답변에 '도움된 답변' 표시 → 문제 해결 후 '해결됨'으로 변경.

**UJ-3 — 실전자료 다운로드 (민지, 프리랜서)**
민지는 실전자료에서 "MCP" 탭 + "Claude Code" 지원환경 필터 → 카드에서 평점·다운로드 수 확인 → 상세에서 사용법·주의사항 확인 → zip 다운로드 → 적용 후 후기 댓글·평점 작성.

**UJ-4 — 자료 기여 (현우, 회원)**
현우는 자기 Cursor Rules를 zip으로 등록(자료명·유형·지원환경·설명·태그 입력) → 검수 없이 즉시 공개 → 다운로드·평점·후기 누적.

**UJ-5 — 운영자 콘텐츠/모더레이션 (운영자)**
운영자는 핵심 가이드 글 작성 → 메인 노출/상단 고정 지정 → 신고 큐 확인 → 부적절 자료 숨김 → 통계로 검색 유입·다운로드 추이 확인.

## 6. 기능 요구사항 (Functional Requirements)

> FR ID는 전역 고유·고정. 그룹은 읽기 편의를 위한 묶음.

### 6-1. 계정 & 인증 (FR-1.x)
- **FR-1.1** 회원가입: 이메일+비밀번호. `[ASSUMPTION]` 가입 시 이메일 인증. 닉네임은 사용자 입력이 아니라 시스템 자동 배정(FR-1.10).
- **FR-1.2** **소셜 로그인: 구글 / 네이버 / 카카오** (사용자 확정). 최초 소셜 로그인 시에도 닉네임·기본 프로필 이미지는 자동 배정(FR-1.10), 이후 수정 가능.
- **FR-1.3** 로그인 / 로그아웃, 세션 유지.
- **FR-1.4** 비밀번호 해시 저장, 비밀번호 재설정(재설정 메일 발송).
- **FR-1.5** **공개 프로필 페이지**: 닉네임·소개·프로필 이미지·배너·작성 글/답변·등급·뱃지·팔로워/팔로잉 수·팔로우 버튼(JSON-LD ProfilePage, FR-11.5 연계). 팔로우 실동작은 FR-7.10. `[ASSUMPTION]` 공개 범위·표시 항목(Open Q-9).
- **FR-1.6** **마이페이지(`/me` — 활동 허브)**: 요약(등급·포인트·뱃지), **내 활동**(글/질문/답변/**댓글**/등록 자료), 북마크, **내 뱃지·등급**, 알림, 쪽지, **1:1 문의** 진입. 로그인 필요·noindex. *(마이페이지 구조 = Open Q-9 확정, 아키텍처 단계.)*
- **FR-1.9** **계정 설정(`/settings`)**: **회원정보 수정**(닉네임·소개·프로필 이미지·배너 이미지·외부 링크), **비밀번호 변경**(로그인 상태 — FR-1.4의 분실 재설정과 별개), 알림 설정(FR-12.3 연계), 차단 목록(FR-7.9 연계), 회원 탈퇴(FR-1.7 연계). 민감 영역이라 마이페이지와 분리. 닉네임 변경 시 유니크 재검증, 커스텀 이미지 업로드 시 기본 이미지를 덮어씀.
- **FR-1.7** **회원 탈퇴/계정 삭제** + 작성 콘텐츠 처리 정책. `[ASSUMPTION]` 탈퇴 시 글 익명화 vs 삭제(Open Q-12).
- **FR-1.8** 비회원/회원/운영자 권한 분기(5-1 역할표 기준).
- **FR-1.10** **자동 닉네임·기본 프로필 이미지**: 가입(이메일/소셜) 즉시 시스템이 유니크 닉네임(형용사+명사+숫자 한국어 조합, 충돌 시 재시도)과 기본 프로필 이미지(준비된 N종 중 랜덤)를 자동 배정. 사용자는 이후 계정 설정에서 변경(FR-1.9). 기본 이미지는 인덱스로 보관해 "미변경=기본" 구분. *(기획 추가 2026-06-22.)*

### 6-2. 공통 게시판 (FR-2.x)
적용 게시판: 바이브코딩 가이드/팁, 자동화 가이드/사례/팁, 외주·판매 팁, 수익화 사례, AI 창작마당, 내가 만든 AI 제품.
- **FR-2.1** 게시판 **목록(텍스트형 리스트)**: 게시판명·설명, 검색창, 정렬 탭(전체/인기글/최신글/댓글 많은 글), 글쓰기 버튼, 글 항목(제목·요약·태그·작성자·작성일·조회수·댓글수·좋아요수·이미지/링크 첨부 아이콘), 페이지네이션. 목록에 썸네일 미사용.
- **FR-2.2** 게시글 **상세**: 게시판명·제목·작성자·작성일·조회수·댓글수·좋아요수·태그·본문·이미지/링크/코드블록/첨부파일 표시·좋아요·공유·신고·댓글·수정/삭제·목록으로.
- **FR-2.3** 본문 렌더링 안전성: 줄바꿈/들여쓰기 보존, 이미지 본문 내 표시(클릭 확대), 링크 자동 인식, 코드블록 보존, **HTML/script 실행 차단(XSS 방지)**.
- **FR-2.4** 게시글 **글쓰기**: 제목·본문 에디터·태그·이미지 첨부·외부 링크·파일 첨부·임시저장·등록/취소. 게시판은 진입 경로 기준 자동 지정. 회원에게 SEO 입력 노출 안 함.

**설계 철학(확정)**: "가벼운 명예 중심". 포인트는 **뒷단 점수**(노출 최소)이고, 사용자에게 보이는 건 **등급·뱃지·랭킹**(인정/신뢰 표시)이다. **글쓰기 등 핵심 활동에 포인트/등급 제한을 두지 않는다.** 포인트는 '동기'가 아니라 등급·뱃지를 계산하는 '회계 장부' 역할.

- **FR-9.1 포인트(뒷단 점수)** — 활동 기반 적립. 초기 기준값(*튜닝 대상*):
  - 실전자료 등록 +20 / 글 작성 +10 / 답변 작성 +5 / 댓글 +1
  - 받은 좋아요(글·답변·자료) +2/건 / 내 자료 다운로드 +1/건(1인 1자료 1회, 일일 상한)
  - **어뷰징 방지**: 자가 좋아요 불가, 콘텐츠 삭제/숨김 시 포인트 회수, 일일 적립 상한, 반복 등록 차단(NFR-2).
  - 포인트 숫자는 마이페이지에서만 작게 노출(랭킹/등급 산정용).
- **FR-9.2 등급** — 누적 포인트 기반 5단계, 이름 옆 + 프로필에 뱃지 표시. **아무 기능도 잠그지 않음**. 초기 단계(*튜닝 대상*):
  - Lv1 입문 작당러(0+) / Lv2 작당러(100+) / Lv3 실전 작당러(500+) / Lv4 고수 작당러(1500+) / Lv5 작당 마스터(4000+)
- **FR-9.3 뱃지(성취 인정)** — 초기 목록(*확장 가능*): 첫 발걸음(첫 글), 자료 기여자(첫 실전자료), 인기 자료(자료 다운로드 100+), 인기글(글 좋아요 50+), 답변러(답변 N개), 꾸준러(N주 연속 활동), + **운영자 수여 특별 뱃지**.
- **FR-9.4 랭킹** — **주간/월간 기여자 TOP**(기간 내 포인트 증가량 기준). 메인/사이드 코너 위젯 + 별도 랭킹 페이지. 초기 모수 적을 땐 과도 노출 자제.
- **충돌 해소(확정)**: 묻고답하기 '도움된 답변'은 포인트/보상과 **연결하지 않음**(FR-3.7, 부담 없는 표시 유지). 대신 **답변이 받은 '좋아요'로 포인트 적립** → 사적 표시(도움된 답변)와 공개 인정(좋아요)을 분리.

## 7. 비기능 요구사항 (NFR)

- **NFR-1 (SEO 렌더링)**: 공개 페이지는 **서버 렌더링(SSR)**으로 검색엔진/AI 검색이 본문을 즉시 파싱 가능해야 한다. 의미론적 마크업(시맨틱 HTML) 준수. *(본 제품 #1 과제와 직결)*
- **NFR-2 (보안)**: 사용자 입력 본문의 HTML/script 실행 차단(XSS). 업로드 파일은 허용 확장자만, 안전한 저장/서빙. **업로드 파일 자동 보안 검사**(악성코드 스캔 — 사용자 확정 Phase 1). API Key 등 민감정보 노출 경고 문구. 비밀번호 해시. CSRF·rate limiting·어뷰징 방지(게이미피케이션 자가 추천/반복 등록 차단).
- **NFR-3 (반응형/모바일)**: 전 화면 모바일 대응. 실전자료 목록은 모바일에서 필터 접힘 + 카드 버튼 2개([다운로드][상세보기]), 상세는 다운로드 하단 고정 버튼. 표 기능 미제공(가독성).
- **NFR-4 (성능)**: 목록/상세/검색 응답이 커뮤니티 사용에 충분히 빠를 것. `[ASSUMPTION]` 구체 수치 목표 미정(Open Q-11).
- **NFR-5 (접근성)**: 이미지 alt, 명확한 헤딩 구조(SEO와 공유).
- **NFR-6 (가용성/백그라운드)**: 이메일 발송·이미지 변환 등 무거운 작업은 백그라운드 처리(서비스 응답 지연 방지).
- **NFR-7 (확장성)**: 인증/타입/검증/비즈니스 로직을 향후 React Native 앱과 공유 가능하도록 설계(상세 addendum).
- **NFR-8 (URL 안정성)**: URL은 초기부터 고정 지향(검색 유입 보호). 구조는 8장.

## 11. 용어 (Glossary)
- **공통 게시판**: 텍스트형 리스트 구조를 쓰는 일반 게시판(바이브코딩·자동화·수익화·라운지 등). 묻고답하기/실전자료는 별도 구조.
- **묻고답하기**: 통합 질문 공간. 질문=게시글형, 답변=확장 댓글형.
- **도움된 답변**: 질문 작성자가 참고한 답변 1개에 다는 약한 표시. 채택/정답/보상 아님.
- **상태값**: 묻고답하기 질문의 상태 — 답변대기 / 답변있음 / 해결됨.
- **실전자료**: 다운로드형 자료실 콘텐츠(프롬프트·Claude Code Skill·MCP·Rules·설정·템플릿·체크리스트).
- **대표 다운로드 파일**: 실전자료 첨부 중 다운로드 수 집계 기준이 되는 파일 1개.
- **작당러 / 등급**: 회원 등급 명칭(입문 작당러~작당 마스터). 명예 표시이며 기능을 잠그지 않음.
- **포인트**: 등급·랭킹 산정용 뒷단 점수. 화면 전면 노출 안 함.
- **행동(게이팅 대상)**: 다운로드·좋아요·북마크·댓글·평점·후기·쪽지·신고 — 로그인 필요. 읽기·검색은 비회원 개방.
