# 스토리 교정 스펙 (사용자 결정 반영)

이미 생성된 스토리 파일에 대해 **사용자가 확정한 결정**을 반영해 교정한다.
아래 "정규 규칙"을 위반하는 모든 서술(AC·Tasks·Dev Notes·References·파일경로·라우트)을 찾아 고친다.
**블라인드 치환 금지** — 맥락을 읽고 의미가 맞게 고쳐라. 스토리의 다른 정상 내용은 보존.

응답·문서 언어: 한국어. `Status: ready-for-dev` 유지.

---

## 정규 라우트/구조 규칙 (이게 진실. 기존 코드 기준)

1. **마이페이지 활동 허브 = `/mypage`** (절대 `/me` 아님).
   - 기존 `apps/web/app/mypage/page.tsx`는 이미 탭 구조: `posts`(내가 쓴 글)·`comments`(댓글)·`bookmarks`(북마크)·`likes`(좋아요)·`following`(팔로잉)·`followers`(팔로워).
   - `/me`, `/me/activity`, `/me/badges`, `/me/bookmarks` 등 **신규 서브라우트 생성 금지**. `/mypage`→`/me` 리다이렉트 같은 이전 작업 **금지**.
   - 활동(글/댓글/북마크/좋아요/팔로잉/팔로워)은 `/mypage`의 **탭**으로 처리.
2. **알림 = `/notifications`** (최상위 독립, 기존 존재). `/me/notifications` 아님.
3. **쪽지 = `/messages`** (최상위 독립, 기존 존재). `/me/messages` 아님.
4. **설정 = `/settings/*`** (기존: `/settings/profile`·`/settings/security`·`/settings/notifications`). 마이페이지와 분리 유지.
5. **1:1 문의(운영진 고객지원 문의) = 별도 독립 라우트** (디자인 신규 생성 필요):
   - 유저: `/inquiries` — 문의하기(작성) + 내 문의 내역(목록·상세·운영진 답변 스레드). `/me/inquiries` 아님.
   - 어드민: 신규 "문의 관리" 메뉴(`apps/admin/app/inquiries/`).
   - **중요: 1:1 문의는 디자인 페이지가 아예 없다.** 해당 스토리(유저 7.5 / 어드민 9.14)는 기존 디자인 시스템 컴포넌트(Button·Badge·Card·폼 등)로 **새 화면 레이아웃을 직접 구성**하는 작업을 Tasks에 포함해야 한다(단순 API 연동이 아님).
6. **Q&A = 페이지 `/questions`, API `/api/v1/qna/*`** — 이미 올바름. 변경 금지(참고용. Epic 3는 교정 대상 아님).
7. **약관/정책 = `/terms`(이용약관)·`/privacy`(개인정보처리방침)·`/operation-policy`(운영정책)** (디자인 신규 생성 필요):
   - `/legal/*` 경로 전부 폐기 → 위 경로로 교체.
   - **약관도 디자인 페이지가 없다.** Epic 10 스토리는 기존 디자인 시스템으로 약관 본문 레이아웃(목차·섹션·개정일 등) 화면을 **직접 구성**하는 작업을 포함.
   - 푸터 링크·가입폼 동의 링크 전부 `/terms`·`/privacy`·`/operation-policy`로.
8. **실전자료 목록 = 4개 독립 페이지 유지** (`/resources/prompts`·`/resources/mcp-skills`·`/resources/rules`·`/resources/templates`).
   - 통합 `/resources` + 유형 탭으로 재편 **금지**. 각 페이지는 자기 자료유형으로 필터된 카드형 목록.
   - `/resources`(인덱스/랜딩)는 기존대로 두되, 유형별 진입은 4개 독립 페이지.
9. **실전자료 등록 폼 = 7-Step 구조** (epics 방식): 유형 선택 → 공통 정보 → 첨부파일 → 사용법/주의사항 → 태그 → 미리보기 → 등록. (기존 단일 스크롤 폼을 7-Step으로 재구성.)
10. **어드민 운영자 계정 관리 = `/admin-members`** (기존 경로 유지). `/admin-accounts`로 **rename 금지** — 모든 `/admin-accounts` 언급을 `/admin-members`로.
11. **댓글 dislike(싫어요) 버튼 = 유지** (제거 금지). 기존 CommentItem의 like/dislike 양쪽 UI 그대로. "dislike 제거" 류 지시는 삭제.
12. **신고 사유 = 기존 ReportModal 그대로 유지**: 스팸 / 욕설 / 개인정보 노출 / 허위정보 / 기타.
    - epics의 다른 enum(음란물·광고·저작권 추가)으로 **변경 금지**. `reasonCode`를 epics 기준으로 바꾸라는 지시는 기존 사유 기준으로 되돌린다.
13. **프로필 이미지·배너 용어/방식**:
    - "아바타"는 곧 프로필 이미지 — FR-1.10대로 가입 시 기본 이미지 자동 배정, 사용자가 설정에서 변경. ("아바타가 따로 있다"는 식의 오해 표현이 있으면 프로필 이미지로 정리.)
    - 배너 = 유저 프로필 페이지 상단 배너. 기본 이미지 제공 + 사용자가 나중에 변경.
    - 커스텀 이미지 업로드 방식은 **API 서버 업로드(멀티파트) 기본**으로 명시(presigned 강제 아님). 이 디테일로 스토리가 모호하면 서버 업로드로 확정.

## 작업 지침

- 너에게 지정된 Epic의 스토리 파일들만 연다.
- 각 파일에서 위 규칙에 어긋나는 부분을 수정한다. 라우트 경로, 신규 파일 경로(`apps/web/app/me/...` → 올바른 경로), Tasks 항목, Dev Notes 설명, References, AC 문구 모두 포함.
- 1:1 문의(7.5/9.14)·약관(10.x) 스토리는 **디자인 페이지 신규 생성**이 작업에 포함되도록 Tasks/Dev Notes를 보강한다.
- 수정 후 `Status: ready-for-dev` 유지. sprint-status.yaml은 건드리지 마라(키 = 파일명이므로 파일명도 바꾸지 마라).
- 완료 시: 수정한 파일 목록 + 각 파일에서 적용한 변경 요약(불릿)만 간결히 반환.
