# AI작당 관리자 페이지 기획서

문서 작성일: 2026-06-15  
문서 목적: AI작당 커뮤니티 사이트의 관리자 페이지 기능, 메뉴 구조, 운영 기준, 개발 우선순위를 정리한 기준 문서

---

## 1. 관리자 페이지 기본 방향

AI작당의 관리자 페이지는 단순히 게시글을 삭제하거나 회원을 관리하는 화면이 아니다.

AI작당은 바이브 코딩, AI 자동화, AI 수익화, 실전자료 다운로드를 중심으로 운영되는 실전형 AI 커뮤니티이므로 관리자 페이지도 다음 목적을 가져야 한다.

```text
1. 커뮤니티 운영 관리
2. 콘텐츠 품질 관리
3. 신고 및 위험 자료 사후관리
4. 회원 활동 관리
5. 포인트 / 등급 / 뱃지 기반 참여 유도
6. 광고 및 수익화 관리
7. 접속 통계 기반 마케팅 분석
```

핵심 기준은 다음이다.

```text
관리자 페이지는 단순 관리용이 아니라
운영 + 마케팅 + 수익화 판단 도구로 만든다.
```


## 3. 개발 우선순위

초기 개발에서 모든 기능을 같은 깊이로 만들면 개발량이 과도하게 커진다.

따라서 메뉴는 모두 포함하되, 기능 깊이는 우선순위에 따라 나눈다.

### 3-1. 1차 개발 핵심 포함

```text
- 대시보드
- 접속 통계
- 게시글 관리
- 묻고답하기 관리
- 실전자료 관리
- 댓글·후기 관리
- 신고 관리
- 회원 관리
- 광고 관리
```

### 3-2. 1차 개발 기본형 포함

```text
- 포인트 관리
- 등급 관리
- 뱃지 관리
```

포인트, 등급, 뱃지는 커뮤니티 활성화에 필요하지만 처음부터 너무 복잡하게 만들면 운영 부담이 커진다.

초기에는 다음 수준으로 시작한다.

```text
포인트 관리
- 포인트 적립 규칙
- 회원별 포인트 내역
- 수동 지급 / 차감

등급 관리
- 등급 목록
- 등급 기준
- 포인트 기준 자동 등급
- 수동 등급 변경

뱃지 관리
- 뱃지 목록
- 수동 지급
- 일부 자동 지급 조건
```

## 4. 대시보드

대시보드는 관리자가 사이트 상황을 빠르게 파악하는 첫 화면이다.

대시보드는 예쁜 차트 중심이 아니라, 오늘 무엇을 봐야 하는지 알려주는 운영 중심 화면이어야 한다.

### 4-1. 대시보드 핵심 지표

```text
오늘 방문자 수
어제 방문자 수
최근 7일 방문자 수
최근 30일 방문자 수

오늘 페이지뷰
오늘 신규 회원 수
오늘 작성 글 수
오늘 댓글 수
오늘 질문 수
오늘 실전자료 등록 수
오늘 실전자료 다운로드 수
오늘 신고 수
```

### 4-2. 운영 확인 지표

```text
답변대기 질문 수
신고 접수 수
신규 실전자료 수
최근 등록 자료
최근 신고 목록
최근 가입 회원
다운로드 많은 자료
조회수 높은 글
댓글 많은 글
```

### 4-3. 마케팅 확인 지표

```text
검색 유입 많은 페이지
가입 전환이 많은 페이지
자료 다운로드가 많은 페이지
재방문율이 높은 페이지
체류 시간이 긴 콘텐츠
이탈률이 높은 페이지
```

### 4-4. 대시보드 화면 구성 예시

```text
[상단 요약 카드]
방문자 / 페이지뷰 / 신규회원 / 글 / 질문 / 다운로드 / 신고

[운영 알림]
답변대기 질문 / 미처리 신고 / 신고 많은 자료

[콘텐츠 성과]
인기 글 / 인기 질문 / 인기 자료

[마케팅 성과]
검색 유입 상위 페이지 / 가입 전환 상위 페이지 / 다운로드 전환 상위 자료
```

## 5. 접속 통계

접속 통계는 별도 메뉴로 둔다.

AI작당은 검색 유입과 콘텐츠 마케팅이 중요하므로 단순 방문자 수만 보면 안 된다.

접속 통계는 어떤 콘텐츠가 유입을 만들고, 어떤 콘텐츠가 가입과 다운로드로 이어지는지 확인할 수 있어야 한다.

### 5-1. 기본 접속 통계

```text
방문자 수
페이지뷰
세션 수
신규 방문자
재방문자
평균 체류 시간
이탈률
회원 가입 전환 수
글 작성 전환 수
댓글 작성 전환 수
실전자료 다운로드 전환 수
```

### 5-2. 기간 필터

```text
오늘
어제
최근 7일
최근 30일
이번 달
지난 달
사용자 지정 기간
```

### 5-3. 유입 경로 분석

```text
검색 유입
직접 유입
SNS 유입
외부 링크 유입
광고 유입
AI 검색 유입 추정
기타 유입
```

### 5-4. 검색 키워드 분석

```text
유입 검색어
검색어별 방문 수
검색어별 페이지뷰
검색어별 가입 수
검색어별 다운로드 수
검색어별 인기 페이지
검색어별 평균 체류 시간
```

### 5-5. 콘텐츠별 성과 분석

게시글별로 다음 지표를 확인할 수 있게 한다.

```text
조회수
방문자 수
평균 체류 시간
댓글 수
좋아요 수
공유 수
신고 수
검색 유입 수
가입 전환 수
관련 자료 다운로드 수
```

실전자료별로 다음 지표를 확인할 수 있게 한다.

```text
조회수
다운로드 수
다운로드 전환율
평점
후기 수
좋아요 수
신고 수
유입 경로
```

### 5-6. 다운로드 전환율

실전자료에서는 다운로드 전환율이 중요하다.

```text
다운로드 전환율 = 다운로드 수 / 상세 페이지 조회수
```

예시:

```text
조회수 1,000 / 다운로드 20 = 전환율 2%
조회수 300 / 다운로드 80 = 전환율 26.6%
```

후자의 자료가 더 강한 자료다.

### 5-7. 접속 통계에서 중요한 판단

접속 통계를 통해 다음을 판단할 수 있어야 한다.

```text
어떤 글이 검색 유입을 만드는가
어떤 자료가 다운로드를 만드는가
어떤 키워드가 회원가입으로 이어지는가
어떤 게시판이 가장 체류 시간이 긴가
어떤 콘텐츠가 이탈률이 높은가
어떤 주제를 더 써야 하는가
어떤 자료를 메인에 노출해야 하는가
```

## 6. 게시글 관리

게시글 관리는 공통 게시판 글을 관리하는 메뉴다.

대상 게시판은 다음이다.

```text
바이브코딩 가이드
바이브코딩 팁
자동화 가이드
자동화 사례
자동화 팁
외주·판매 팁
수익화 사례
AI 창작마당
내가 만든 AI 제품
```

### 6-1. 목록 기능

```text
게시글 검색
게시판별 필터
작성자 검색
공개 / 숨김 / 삭제 상태 필터
공지글 필터
추천글 필터
메인 노출 필터
작성일 기간 필터
```

### 6-2. 게시글 관리 항목

```text
제목
게시판
작성자
작성일
조회수
댓글 수
좋아요 수
신고 수
상태
공지 여부
상단 고정 여부
추천 여부
메인 노출 여부
```

### 6-3. 관리자 처리 기능

```text
게시글 보기
게시글 수정
공지글 설정 / 해제
상단 고정 설정 / 해제
추천글 지정 / 해제
메인 노출 설정 / 해제
숨김 처리
삭제 처리
복구 처리
```

### 6-4. 회원 글쓰기 화면과 분리할 기능

다음 기능은 관리자 화면에서만 처리한다.

```text
공지글 설정
상단 고정
추천글 지정
메인 노출 여부
숨김 처리
삭제 처리
```

회원 글쓰기 화면에는 관리자용 옵션을 노출하지 않는다.

## 7. 묻고답하기 관리

묻고답하기는 일반 게시판과 구조가 다르므로 별도 관리 메뉴로 둔다.

묻고답하기는 질문 글과 답변 댓글을 함께 관리해야 한다.

### 7-1. 목록 필터

```text
전체
답변대기
답변있음
해결됨
신고 있음
도움된 답변 있음
```

### 7-2. 관리 항목

```text
질문 제목
작성자
작성일
상태
답변 수
조회수
좋아요 수
신고 수
도움된 답변 여부
```

### 7-3. 관리자 처리 기능

```text
질문 보기
질문 수정
질문 숨김
질문 삭제
질문 복구
상태 강제 변경
답변 보기
답변 숨김
답변 삭제
```

### 7-4. 운영자가 하지 않는 기능

운영자는 질문자의 도움된 답변을 대신 지정하지 않는다.

```text
도움된 답변 지정은 질문자 권한으로 유지한다.
```

운영자는 부적절한 질문이나 답변을 관리하는 역할만 한다.

## 8. 실전자료 관리

실전자료 관리는 AI작당 관리자 페이지에서 매우 중요한 메뉴다.

실전자료는 누구나 등록 가능하게 운영하므로, 관리자는 검수자가 아니라 위험한 자료를 사후 관리하는 역할을 한다.

### 8-1. 목록 필터

```text
자료 유형
지원 환경
난이도
공개 / 숨김 / 삭제 상태
신고 여부
작성자
등록일
다운로드 수
평점
```

### 8-2. 관리 항목

```text
자료명
자료 유형
지원 환경
작성자
등록일
업데이트일
다운로드 수
평점
후기 수
신고 수
상태
대표 첨부파일
```

### 8-3. 관리자 처리 기능

```text
자료 상세 보기
자료 수정
자료 숨김
자료 삭제
자료 복구
첨부파일 확인
부적절한 첨부파일 삭제
후기 댓글 관리
신고 내역 확인
```

### 8-4. 실전자료에서 사용하지 않는 관리자 기능

다음 기능은 넣지 않는다.

```text
검수됨
공식 자료
운영자 인증
안전성 보증
자동 설치 테스트
자동 파일 보안 검사
```

이유는 운영자가 자료의 안전성과 작동 여부를 보증하는 구조가 되면 책임이 커지기 때문이다.

## 9. 댓글·후기 관리

댓글·후기 관리는 모든 댓글성 콘텐츠를 한 곳에서 관리하는 메뉴다.

대상은 다음이다.

```text
일반 게시판 댓글
묻고답하기 답변
실전자료 후기
```

### 9-1. 목록 필터

```text
댓글 유형
작성자
작성일
신고 여부
공개 / 숨김 / 삭제 상태
관련 게시판
```

### 9-2. 관리 항목

```text
댓글 내용 일부
댓글 유형
작성자
작성일
관련 게시글
신고 수
상태
```

### 9-3. 관리자 처리 기능

```text
원문 보기
관련 게시글로 이동
댓글 숨김
댓글 삭제
댓글 복구
신고 내역 보기
```

### 9-4. 운영자가 하지 않는 기능

운영자가 회원의 댓글 내용을 직접 수정하지 않는 것이 좋다.

댓글 수정은 신뢰 문제가 생길 수 있으므로, 운영자는 숨김 또는 삭제 중심으로 처리한다.

## 10. 신고 관리

신고 관리는 반드시 통합 관리 화면으로 만든다.

게시글, 댓글, 묻고답하기 답변, 실전자료 신고를 한 화면에서 모아본다.

### 10-1. 신고 대상

```text
게시글
댓글
묻고답하기 질문
```

```text
접수
확인중
```

### 10-4. 관리자 처리 기능

```text
신고 대상 보기
```

### 10-5. 신고 관리 원칙


## 20. 최종 결론

AI작당 관리자 페이지는 단순한 백오피스가 아니다.

AI작당은 콘텐츠 기반 커뮤니티이고, 검색 유입과 실전자료 다운로드가 성장의 핵심이기 때문에 관리자 페이지는 다음 역할을 해야 한다.

```text
1. 문제 있는 콘텐츠를 빠르게 관리한다.
2. 좋은 콘텐츠를 메인과 추천 영역에 노출한다.
3. 어떤 글과 자료가 유입을 만드는지 분석한다.
4. 어떤 키워드가 가입과 다운로드로 이어지는지 확인한다.
5. 광고와 수익화 위치를 관리한다.
6. 포인트, 등급, 뱃지로 회원 활동을 유도한다.
```

따라서 관리자 페이지의 핵심은 다음이다.

```text
운영 관리 + 콘텐츠 관리 + 신고 관리 + 통계 분석 + 광고 관리 + 회원 활성화 관리
```

초기 개발에서는 모든 메뉴를 포함하되, 포인트 / 등급 / 뱃지는 기본형으로 시작하고, 접속 통계와 광고 관리는 마케팅과 수익화에 직접 연결되므로 반드시 MVP에 포함한다.
